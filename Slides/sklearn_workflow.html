Summary of Scikit-learn Tools by Workflow

This is a detailed summary of commonly used classes and functions in Scikit-learn for data preparation - DSEB(MFE).

1. Data Cleaning

Techniques to handle common issues like missing values and outliers.

SimpleImputer

Fills missing values using simple strategies.

class sklearn.impute.SimpleImputer(*, strategy='mean', ...)

Key Parameters:

strategy: 'mean', 'median', 'most_frequent', 'constant'.

fill_value: Value to use when strategy='constant'.

KNNImputer

Fills missing values based on the mean of the K-nearest neighbors.

class sklearn.impute.KNNImputer(*, n_neighbors=5, ...)

Key Parameters:

n_neighbors: Number of neighbors (K) to use.

IterativeImputer

Fills missing values by modeling each feature as a function of other features.

class sklearn.impute.IterativeImputer(*, estimator=None, max_iter=10, ...)

Key Parameters:

estimator: The regressor model to use for prediction.

max_iter: Maximum number of imputation rounds.

LocalOutlierFactor

Identifies outliers based on the local density of a data point's neighborhood.

class sklearn.neighbors.LocalOutlierFactor(*, n_neighbors=20, contamination='auto', ...)

Key Parameters:

n_neighbors: Number of neighbors to consider for each point.

contamination: The expected proportion of outliers in the data.

2. Data Transformation

This module includes tools to scale, encode, and transform input features after cleaning.

StandardScaler

Transforms data to have a mean of 0 and a variance of 1.

class sklearn.preprocessing.StandardScaler()

MinMaxScaler

Scales data to a fixed range, by default [0, 1].

class sklearn.preprocessing.MinMaxScaler(feature_range=(0, 1), ...)

Key Parameters:

feature_range: The desired (min, max) range.

RobustScaler

Scales data using the median and IQR, making it robust to outliers.

class sklearn.preprocessing.RobustScaler(*, quantile_range=(25.0, 75.0), ...)

Key Parameters:

quantile_range: The quantile range to use.

OneHotEncoder

Encodes categorical features (with no ordinal relationship) into binary columns.

class sklearn.preprocessing.OneHotEncoder(*, handle_unknown='error', ...)

Key Parameters:

handle_unknown: 'error' or 'ignore' when new values appear.

['Red']
↓
[1, 0, 0]

OrdinalEncoder

Encodes categorical features (with an ordinal relationship) into integers.

class sklearn.preprocessing.OrdinalEncoder(*, categories='auto', ...)

Key Parameters:

categories: Defines the encoding order for each column.

['High']
↓
[2]

PowerTransformer

Transforms data to make its distribution more Gaussian-like.

class sklearn.preprocessing.PowerTransformer(method='yeo-johnson', ...)

Key Parameters:

method: 'yeo-johnson' (default) or 'box-cox'.

QuantileTransformer

Transforms data to follow a uniform distribution.

class sklearn.preprocessing.QuantileTransformer(*, n_quantiles=1000, output_distribution='uniform', ...)

Key Parameters:

output_distribution: 'uniform' or 'normal'.

n_quantiles: Number of quantiles to use.

KBinsDiscretizer

Bins continuous features into intervals.

class sklearn.preprocessing.KBinsDiscretizer(n_bins=5, *, strategy='quantile', ...)

Key Parameters:

n_bins: The number of bins to produce.

strategy: 'uniform', 'quantile', 'kmeans'.

PolynomialFeatures

Generates polynomial features to capture non-linear relationships.

class sklearn.preprocessing.PolynomialFeatures(degree=2, *, interaction_only=False, ...)

Key Parameters:

degree: The highest degree of the polynomial.

interaction_only: Only create interaction features.

[a, b]
↓
[1, a, b, a², ab, b²]

3. Feature Selection

This module contains algorithms to select the most important subset of features.

VarianceThreshold

Removes features with low or zero variance.

class sklearn.feature_selection.VarianceThreshold(threshold=0.0)

Key Parameters:

threshold: The variance threshold to keep a feature.

SelectKBest

Selects the K best features based on univariate statistical tests.

class sklearn.feature_selection.SelectKBest(score_func=f_classif, *, k=10)

Key Parameters:

score_func: The statistical scoring function (e.g., f_classif, chi2).

k: The number of top features to select.

RFE (Recursive Feature Elimination)

Selects features by recursively removing the weakest features.

class sklearn.feature_selection.RFE(estimator, *, n_features_to_select=None, ...)

Key Parameters:

estimator: The model used to evaluate features.

n_features_to_select: The number of features to select.

RFECV

An advanced version of RFE that automatically finds the optimal number of features via cross-validation.

class sklearn.feature_selection.RFECV(estimator, *, cv=None, scoring=None, ...)

Key Parameters:

estimator: The model to use.

cv: The cross-validation strategy.

scoring: The evaluation metric.

SelectFromModel

Selects features based on their importance from a trained model.

class sklearn.feature_selection.SelectFromModel(estimator, *, threshold=None, ...)

Key Parameters:

estimator: The pre-trained model.

threshold: The threshold for feature selection.

Permutation Importance

Assesses feature importance by measuring the performance drop when a feature's values are randomly permuted.

sklearn.inspection.permutation_importance(estimator, X, y, *, scoring=None, n_repeats=5, ...)

Key Parameters:

scoring: The evaluation metric.

n_repeats: Number of times to repeat the permutation.

4. Workflow, Evaluation & Tuning

Tools to combine all the above steps, split data, tune, and evaluate the final model.

Pipeline

"Packages" a sequence of processing steps and a final model into a single object.

class sklearn.pipeline.Pipeline(steps, ...)

Key Parameters:

steps: A list of (name, transform) tuples.

ColumnTransformer

Applies different transformation steps to different columns of the data.

class sklearn.compose.ColumnTransformer(transformers, ...)

Key Parameters:

transformers: A list of (name, transform, columns) tuples.

train_test_split

Splits the dataset into two parts: a training set and a testing set.

sklearn.model_selection.train_test_split(*arrays, test_size=None, random_state=None, stratify=None)

Key Parameters:

test_size: The proportion of the test set.

random_state: Ensures reproducible splits.

stratify: Preserves class proportions.

cross_val_score

Reliably evaluates the stability and performance of a model.

sklearn.model_selection.cross_val_score(estimator, X, y, *, cv=None, scoring=None)

Key Parameters:

estimator: The model to evaluate.

cv: The number of folds to use.

scoring: The metric to use.

GridSearchCV

Automates the hyperparameter tuning process by trying all parameter combinations.

class sklearn.model_selection.GridSearchCV(estimator, param_grid, ...)

Model Evaluation Metrics (`sklearn.metrics`)

Accuracy

The ratio of correctly predicted instances.

Confusion Matrix

Shows the number of correct and incorrect predictions for each class.

Precision, Recall & F1

More detailed performance metrics for the positive class.

MAE & MSE (Regression)

Measure the average error of predictions.

ROC Curve & AUC

Shows the model's ability to distinguish between classes.
